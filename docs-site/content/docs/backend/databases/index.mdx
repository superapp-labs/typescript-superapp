Every database in `@superapp/backend` is accessed through its native driver. Just pass a connection URL and the engine infers the database type from the protocol.

```typescript
import { createEngine } from '@superapp/backend'

const engine = createEngine({
  connections: {
    main: process.env.PG_URL!,
  },
})
```

## Native vs Custom Providers

**Native providers** connect directly using the database's own driver (`pg`, `mysql2`, `better-sqlite3`). Queries are executed directly against the database with no intermediary.

**Custom providers** implement a `CustomIntegrationProvider` interface for non-database sources -- REST APIs, GraphQL endpoints, edge runtimes, or proprietary systems. Queries are translated into provider-specific calls and results are returned as virtual tables.

## Built-in Providers

| Provider | Connection format | Read | Write | Driver |
|---|---|---|---|---|
| PostgreSQL | `'postgres://...'` | Yes | Yes | `pg` |
| MySQL | `'mysql://...'` | Yes | Yes | `mysql2` |
| SQLite | `'./path/to/file.db'` | Yes | Yes | `better-sqlite3` |
| CSV | `{ directory: './data/' }` | Yes | No | built-in |

## Provider Interface

Every native provider exports a single object that satisfies the `IntegrationProvider` interface:

```typescript
interface IntegrationProvider {
  type: string
  connect: (config: ConnectionConfig) => Promise<DriverConnection>
  disconnect: (connection: DriverConnection) => Promise<void>
  introspect: (connection: DriverConnection) => Promise<TableSchema[]>
  capabilities: {
    read: boolean
    write: boolean
    transactions: boolean
  }
}
```

The `connect` method initializes the native driver connection pool. The engine calls it once on startup (or when a connection is added via the admin UI) and maintains the pool for the lifetime of the process.

## Multiple Connections

Pass all connections in the `connections` object. The engine infers the type from the protocol or format:

```typescript
const engine = createEngine({
  connections: {
    main: process.env.PG_URL!,
    warehouse: process.env.MYSQL_URL!,
    reports: { directory: './data/reports' },
  },
})
```
